## Overview

A lightweight, read-only desktop dashboard for viewing Content Ideas generated by the Autom8Lab Content Engine. Replaces Notion's table/board UI with a purpose-built card grid and slide-out detail panel optimized for scanning and evaluating content ideas.

**This is a personal tool, not a product.** Optimized for one user, one machine, desktop only.

## Scope

### v1 (this spec)
- Read-only Content Ideas view
- Card grid with filters and sorting
- Slide-out detail panel with full idea info (all 5 hooks, score, description, filming setup, etc.)
- Dark mode theme with centralized, swappable design tokens

### v1.1 (future)
- Write operations: status changes, multi-select edits (filming setup, frame type)
- Inline status toggling from cards and detail panel

### v1.2+ (future)
- Morning Menu view (top ideas by score)
- Pipeline status dashboard (counts at each stage)
- Link triage view (quick-set status on transcribed links)
- Filming Today / Ready to Post views

## API Design

### Endpoints

#### `GET /api/ideas`

Returns all content ideas with key fields for the card grid.

**Query params:**
- `status` ‚Äî filter by status, comma-separated for multiple (e.g., `?status=new,queued`)
- `sort` ‚Äî sort field (default: `score`). Valid: `score`, `name`, `status`, `urgency`
- `order` ‚Äî `asc` or `desc` (default: `desc`)
- `filming_setup` ‚Äî filter by filming setup value
- `format` ‚Äî filter by format
- `top_pick` ‚Äî `true` to show only top picks
- `search` ‚Äî text search across name, main_topic, and description (server-side, case-insensitive contains)

**Response:** Array of idea objects with: `id`, `name`, `score`, `top_pick`, `status`, `main_topic`, `format`, `filming_setup` (array), `filming_priority`, `frame_type` (array), `topic_cluster`, `urgency`, `description`

**Pagination:** The Notion API returns max 100 results per request. The API server handles pagination internally, fetching all pages and returning the full list. If the database grows past ~500 ideas, add cursor-based pagination to this endpoint.

#### `GET /api/ideas/{id}`

Returns full detail for a single idea.

**Response:** All fields from the list endpoint plus: `angle`, `reasoning`, `hook_1` through `hook_5`, `original_url`, `source_link` (object: `{ name, url }` ‚Äî resolved from the Notion relation to show the linked page title and its Notion URL), `filmed_date`, `posted_date`, `caption_tiktok`, `caption_instagram`, `caption_youtube`, `caption_linkedin`, `post_urls` (array of strings ‚Äî each a clickable URL where the content was posted)

#### `GET /api/stats`

**Note:** Uses `/api/stats` (not `/api/ideas/stats`) to avoid route conflict with the `{id}` parameter.

Returns summary counts for the header.

**Response:** `{ total, by_status: { new: N, queued: N, ... }, top_picks: N }`

### Data Flow

```
React component
  ‚Üí useIdeas hook (fetch + cache)
    ‚Üí GET localhost:8088/api/ideas
      ‚Üí FastAPI server.py
        ‚Üí Notion API (query Content Ideas DB)
          ‚Üí Response mapped to clean JSON
            ‚Üí Back to React
```

## Component Design

### Layout

- Full-width, dark background
- Fixed header: app name ("Content Ideas"), stats summary line ("47 ideas, 5 top picks"), filter bar
- Below: responsive card grid (3 columns on wide, 2 on narrower desktop)
- Slide-out panel overlays from the right when a card is clicked

### IdeaCard

Compact tile showing at-a-glance info:
- **Score** ‚Äî large, color-coded number (green >= 8, yellow >= 6, red < 6)
- **Name** ‚Äî idea title, truncated if long
- **Top Pick** ‚Äî badge/indicator if flagged
- **Main Topic** ‚Äî text label
- **Format** ‚Äî pill/tag
- **Filming Setup** ‚Äî small tags (e.g., `talking_head`, `screen_recording`)
- **Status** ‚Äî colored pill
- **Filming Priority** ‚Äî color-coded indicator

### IdeaDetail (slide-out panel)

Full detail view, slides in from the right (~40-50% of screen width). Grid stays visible but dimmed behind an overlay. Close via X button or clicking the overlay.

Contents:
- **Header:** Name, Score (large), Top Pick badge, Status pill
- **Section ‚Äî Hooks:** All 5 hooks displayed as a numbered list with clear visual hierarchy. This is the most important section.
- **Section ‚Äî Overview:** Description, Angle, Main Topic, Format, Urgency
- **Section ‚Äî Reasoning:** The creative director's scoring rationale
- **Section ‚Äî Filming:** Filming Setup tags, Filming Priority, Frame Type tags
- **Section ‚Äî Meta:** Topic Cluster, Original URL (clickable), Source Link info
- **Section ‚Äî Captions** (if present): TikTok, Instagram, YouTube, LinkedIn captions
- **Section ‚Äî Dates** (if present): filmed_date, posted_date, post URLs

### FilterBar

Top of page, horizontal layout:
- **Sort by:** dropdown ‚Äî Score (default), Name, Status, Urgency
- **Filter by Status:** multi-select pills ‚Äî new, queued, filming_today, filmed, captioned, posted, archived
- **Filter by Filming Setup:** multi-select
- **Filter by Format:** dropdown
- **Top Picks only:** toggle
- **Search:** text input filtering across Name, Main Topic, Description

## Serving Model

### Development
1. Start FastAPI: `cd api && uvicorn server:app --port 8088`
2. Start Vite: `cd frontend && npm run dev`
3. Open `localhost:5173` ‚Äî Vite proxies `/api` to `8088`

1. Build frontend: `cd frontend && npm run build`
2. Start FastAPI: `cd api && uvicorn server:app --port 8088`
3. Open `localhost:8088` ‚Äî FastAPI serves built React app + API

## UI States

- **Loading:** Centered spinner on first load. Inline skeleton cards on subsequent fetches.
- **Error:** Banner at top of page with error message (e.g., "Could not reach API" or "Notion API error"). Cards from last successful fetch remain visible if available.
- **Empty:** Centered message ‚Äî "No ideas match your filters" with a clear filters button.

## Constraints & Decisions

1. **Desktop only** ‚Äî no responsive mobile breakpoints, optimized for wide screens
2. **Read-only for v1** ‚Äî no write operations to Notion, reduces complexity and API surface
3. **No authentication** ‚Äî local machine only, no need
4. **No caching layer** ‚Äî Notion API is fast enough for one user. Can add if it becomes slow.
5. **No WebSocket/real-time** ‚Äî manual refresh is fine for a personal tool
6. **styled-components for theming** ‚Äî built-in ThemeProvider, good DX with theme tokens, component-scoped
7. **Route ordering** ‚Äî `/api/stats` registered before `/api/ideas/{id}` in FastAPI to avoid path conflict
